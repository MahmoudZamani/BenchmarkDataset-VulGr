VulGr: Benchmark Graph Dataset for Source and Binary Vulnerability Detection Systems

VulGr is a new graph vulnerability detection dataset based on a novel graph structure that combines source- and binary-level code features into a composite graph to expose vulnerabilities in real-world applications. Recent works have applied GNNs to reduce the human effort of detecting vulnerabilities in large codes by converting source code information into a graph for GNN training. Such approaches convert source codes to intermediate representations, from which Control-Flow Graphs (CFGs) and Abstract Syntax Trees (ASTs) are extracted. Binary graph representations can also be used with GNNs, but at the expense of losing many important features, such as data dependencies and control-flows only available from sources. However, many real-world software attacks exploit a combination of source-level bugs and binary-level implementation details, and are therefore difficult or impossible to detect using source- or binary-level features alone. To overcome this issue, VulGR introduces a novel composite Cross-Domain Code Property Graph (CDCPG) representation that leverages the vulnerability pattern recognition of GNNs by combining these two different domains. Graphs from six different C/C++ applications are accumulated into the VulGr dataset, with each graph labeled vulnerable or non-vulnerable using information drawn from public vulnerability databases. The benchmark performance of popular GNN methods in this domain are then evaluated, with the best performing model yielding an F1-score of 0.855 and Recall of 0.908.
